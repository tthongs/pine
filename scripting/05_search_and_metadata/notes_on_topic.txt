# Notes on Topic: Search and Metadata

## 📖 Overview
The `find` command is the primary tool in Bash for discovering files based on their metadata (name, size, permissions, time) rather than their content. It allows for complex filtering and immediate execution of commands on the discovered items.

## 🛠️ Commands and References

### 1. Basic Filtering

| Option | Description | Example |
| :--- | :--- | :--- |
| `-name` | Case-sensitive name pattern. | `find . -name "*.sh"` |
| `-iname` | Case-insensitive name pattern. | `find . -iname "README.md"` |
| `-type f` | Search for regular files only. | `find /var/log -type f` |
| `-type d` | Search for directories only. | `find /home -type d` |

### 2. Metadata Filtering

| Option | Description | Example |
| :--- | :--- | :--- |
| `-maxdepth N` | Limit search to N levels of depth. | `find . -maxdepth 1` |
| `-perm <mode>` | Filter by octal permissions. | `find . -perm 755` |
| `-size <size>` | Filter by size (+/- for range). | `find . -size +10M` |
| `-mtime <days>` | Filter by modification time (in days). | `find . -mtime -7` (last 7 days) |

### 3. Dynamic Execution (`-exec`)
The `-exec` flag allows you to run a command on every file that matches the criteria.

| Syntax | Description | Performance |
| :--- | :--- | :--- |
| `-exec cmd {} \;` | Runs `cmd` once for EVERY file. | Slower (spawns many processes). |
| `-exec cmd {} +` | Passes multiple files as arguments to one `cmd`. | Faster (efficient batching). |
| `-ok cmd {} \;` | Same as `-exec`, but prompts for confirmation. | Safest (interactive). |

**Pro Tip: Basename Extraction** While `-exec basename {} \;` works, using `-printf "%f\n"` is a built-in GNU find feature that is significantly faster because it doesn't spawn an external process.

## 💡 Practical Use-Cases
1. **Cleanup Old Logs:** `find /var/log -name "*.log" -mtime +30 -exec rm {} \;` deletes logs older than 30 days.

2. **Permission Auditing:** `find . -type f -perm 777` identifies dangerously over-privileged files.

3. **Batch Renaming/Processing:** Using `-exec bash -c '...'` to perform complex logic on each file found.
