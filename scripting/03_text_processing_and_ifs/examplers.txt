# Examplers: Text Processing and IFS

### Question 1: The Subshell Pitfall
Why does the variable `count` remain `0` after the following loop, and how do you fix it using IFS or redirection?
```bash
count=0
echo -e "one\ntwo\nthree" | while read line; do
    ((count++))
done
echo "Total: $count"
```

**Answer:**
The pipe (`|`) creates a **subshell** for the `while` loop. Changes to variables inside a subshell are lost when the subshell terminates.
**Fix (using Process Substitution):**
```bash
count=0
while read line; do
    ((count++))
done < <(echo -e "one\ntwo\nthree")
echo "Total: $count"
```

### Question 2: Custom Delimiter Parsing
Given the string `data="server01|192.168.1.1|UP"`, write a single command to extract the IP address and status into two variables named `ip` and `status`.

**Answer:**
```bash
IFS="|" read -r junk ip status <<< "$data"
```

**Explanation:**
- `IFS="|"` tells `read` to split on the pipe character.
- `junk` captures the first field (`server01`).
- `ip` and `status` capture the second and third fields respectively.
- Using `<<< "$data"` (a "Here String") feeds the variable into the `read` command.

### Question 3: Regex Capture Groups
Write a script snippet that checks if a filename matches the pattern `Report_YYYY-MM.pdf` and, if it does, prints only the Year and Month.

**Answer:**
```bash
filename="Report_2026-03.pdf"
regex='^Report_([0-9]{4})-([0-9]{2})\.pdf$'

if [[ $filename =~ $regex ]]; then
    echo "Year: ${BASH_REMATCH[1]}"
    echo "Month: ${BASH_REMATCH[2]}"
fi
```

**Explanation:**
- `([0-9]{4})` captures exactly 4 digits into `BASH_REMATCH[1]`.
- `([0-9]{2})` captures exactly 2 digits into `BASH_REMATCH[2]`.
- The `^` and `$` anchors ensure the pattern matches the *entire* filename, preventing partial matches like `MyReport_2026-03.pdf.bak`.
