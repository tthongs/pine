# Notes on Topic: Arrays

## 📖 Overview
Arrays in Bash allow you to store multiple values in a single variable. Bash
supports both **Indexed Arrays** (numeric keys) and **Associative Arrays**
(string keys).

## 🛠️ Commands and References

### 1. Indexed Arrays
- **Declaration:** `fruits=("Apple" "Banana" "Cherry")`

- **Access:** `${fruits[0]}`

- **Length:** `${#fruits[@]}`

- **All Elements:** `"${fruits[@]}"` (Always use quotes!)

- **Indices:** `"${!fruits[@]}"`

### 2. Associative Arrays (Bash 4.0+)
- **Declaration:** `declare -A user_roles` (Mandatory)

- **Assignment:** `user_roles[alice]="admin"`

- **Access:** `${user_roles[alice]}`

### 3. Common Operations

| Operation | Syntax | Description |
| :--- | :--- | :--- |
| **Append** | `arr+=("NewVal")` | Adds to the end of the array. |
| **Delete** | `unset 'arr[1]'` | Removes element at index 1. |
| **Slicing** | `${arr[@]:1:2}` | Gets 2 elements starting at index 1. |
| **Search/Replace** | `${arr[@]/pat/rep}` | Replaces pattern in all elements. |
| **Re-indexing** | `arr=("${arr[@]}")` | Closes gaps after an `unset`. |

### 4. The Subshell Pitfall
Piping data to a `while` loop to populate an array will fail because the loop
runs in a subshell.

- **Wrong:** `cat file | while read; do arr+=($REPLY); done` (Array will be
empty).

- **Correct:** `while read; do arr+=($REPLY); done < <(cat file)` (Process
substitution).

## 💡 Practical Use-Cases
1. **Batch File Processing:** Storing a list of files found by `find` into an
array for sequential processing.

2. **Config Mapping:** Using associative arrays to map environment names to
their respective IP addresses.

3. **Data Filtering:** Iterating through an array and creating a new "filtered"
array based on specific criteria.
