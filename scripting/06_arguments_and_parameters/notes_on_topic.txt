# Notes on Topic: Arguments and Parameters

## 📖 Overview
Arguments are the primary way to pass external data into a Bash script. This topic covers positional parameters, tracking argument counts, and shifting parameters for sequential processing.

## 🛠️ Commands and References

### 1. Positional Parameters
Bash automatically assigns arguments to special variables based on their position.

| Variable | Description |
| :--- | :--- |
| `$0` | The name of the script being executed. |
| `$1, $2, ... $n` | The first, second, through n-th arguments. |
| `$#` | The total number of arguments passed to the script. |
| `$@` | All arguments as a quoted list (`"$1" "$2" ...`). |
| `$*` | All arguments as a single string (`"$1 $2 ..."`). |

### 2. Manual Parameter Management
- **`set -- arg1 arg2 ...`**: Overwrites the current positional parameters with the provided values. This is useful for testing or re-parsing data within a script.

- **`shift [n]`**: "Consumes" the first `n` arguments (default 1) and shifts all remaining arguments to the left. `$2` becomes `$1`, `$3` becomes `$2`, and `$#` decreases accordingly.

### 3. Parsing with `getopts`
The `getopts` built-in provides a robust way to parse short, single-character 
flags in Bash.

- **Basic Usage:** `while getopts 'uh' opt; do` parses `-u` and `-h`.
- **Arguments:** Appending a colon (e.g., `f:`) means the flag requires a 
  value. The value is captured in the `$OPTARG` variable.
- **Silent Errors:** Starting the option string with a colon (e.g., `:hf:`) 
  suppresses default Bash error messages.
- **Handling Errors:**
  - `\?)`: Triggered when an invalid option is provided. The invalid character 
    is stored in `$OPTARG`.
  - `:)`: Triggered when a required argument is missing for an option. The 
    option character is stored in `$OPTARG`.
- **Argument Validation:** It is a best practice to use regex (e.g., 
  `[[ $OPTARG =~ ^(-h|-l)$ ]]`) to ensure that an argument passed to a flag 
  isn't accidentally another flag due to a user omission.

## 💡 Practical Use-Cases
1. **Argument Validation:** Checking `if [[ $# -lt 2 ]]` to ensure the user provided enough information before proceeding.

2. **Sequential Processing:** Using a `while` loop with `shift` to process a list of files passed as arguments one by one.

3. **Usage Menus:** Passing the script name (`$0`) to a help function to ensure the correct name is displayed in the instructions.
