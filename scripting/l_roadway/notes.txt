## 1. UNIX Fundamentals & Navigation
The foundation of everything you do in the terminal.

### **Essential Commands**
- `pwd`: Print Working Directory (Absolute path starting from `/`).
- `ls`: List files.
  - `-l`: Long list (permissions, size, owner).
  - `-a` / `-A`: Show hidden files (starting with `.`).
  - `-F`: Append indicators (`/` for dirs, `*` for executables).
  - `-ld`: List directory info itself, not its contents.
- `mkdir -p D1/D2`: Create nested directories.
- `cp -r src dest`: Recursive copy (for directories).
- `rm -rf`: Force recursive remove. **No undo.**

### **Path Logic**
- `/`: The Root (top-level).
- `~`: Your Home Directory (`/home/pine`).
- `.`: Current directory.
- `..`: Parent directory.
- `*`: Wildcard (matches zero or more characters).

## 2. Permissions & Security
Understanding who can do what to a file.

### **Permission Groups**
`[Type][Owner][Group][Others]`
Example: `-rw-rw-r--`
- `-`: Regular file.
- `d`: Directory.
- `l`: Link.

### **Octal Math (Binary Logic)**
- `r (Read)` = 4
- `w (Write)` = 2
- `x (Execute)` = 1
- **777**: Everyone can do everything.
- **644**: Owner (rw), Group (r), Others (r).
- **755**: Owner (rwx), Group (rx), Others (rx).

### **umask (Default Permissions)**
- New Files default to `666 - umask`.
- New Dirs default to `777 - umask`.
- If `umask` is `002`, a new file gets `664`.

## 3. The Power of the GVIM Editor
A master's tool for editing code inside the terminal.

### **Modes**
- **Command Mode (CM):** Default mode for navigation and editing commands.
- **Insert Mode (IM):** For typing text. Press `i`, `a`, or `o` to enter.
- **Esc:** Always returns you to Command Mode.

### **Navigation (Command Mode)**
- `h`, `j`, `k`, `l`: Left, Down, Up, Right.
- `w`: Jump forward by word.
- `b`: Jump backward by word.
- `0`: Start of line.
- `$`: End of line.
- `gg`: Top of file.
- `G`: Bottom of file.

### **Editing (Command Mode)**
- `x`: Delete single character.
- `dd`: Delete (cut) current line.
- `yy`: Yank (copy) current line.
- `p`: Paste below cursor.
- `u`: Undo last action.
- `cc`: Change line (delete + enter Insert Mode).
- `:set nu`: Enable line numbers.
- `:w!`: Force save.
- `:wq`: Save and quit.

## 4. Core Text Processing Utilities
The "Swiss Army Knives" of the command line.

### **wc (Word Count)**
- `-l`: Lines.
- `-w`: Words.
- `-m`: Characters.

### **cut (Column Extraction)**
- `-d','`: Set delimiter (comma).
- `-f1,3`: Extract fields 1 and 3.
- `-c2-5`: Extract character range 2 to 5.

### **tr (Translate/Delete)**
- `tr 'a-z' 'A-Z'`: Convert to uppercase.
- `tr -d ' '`: Delete all spaces.
- `tr -s ' '`: Squeeze multiple spaces into one.

### **Head & Tail**
- `head -n 5`: First 5 lines.
- `tail -n 2`: Last 2 lines.

## 5. Shell Mechanics & Environment
How the shell interprets your commands.

### **Quotes**
- **Weak Quotes (`" "`)**: Variables are expanded (`"$USER"` $\rightarrow$ `pine`).
- **Strong Quotes (`' '`)**: Everything is literal (`'$USER'` $\rightarrow$ `$USER`).
- **Escape (`\`)**: Escapes the next character (e.g., `\$`).

### **Variables**
- **User Defined:** `v=100` (No spaces around `=`).
- **Environment:** `$SHELL`, `$PATH`, `$USER`, `$HOSTNAME`.
- **Command Substitution:** `file=$(ls)` or `file=`ls``. Stores command output in a variable.

### **Redirection (I/O)**
- `0 <`: Standard Input (stdin).
- `1 >`: Standard Output (stdout - overwrite).
- `1 >>`: Standard Output (stdout - append).
- `2 >`: Standard Error (stderr).
- `&>`: Both stdout and stderr.
- `|`: Pipe (Output of cmd1 $\rightarrow$ Input of cmd2).

## 7. Process Management
How to run and monitor programs.

- **PID (Process ID):** Every command gets a unique ID.
- **&**: Run command in the background (`gvim file &`).
- **jobs -l**: List background jobs in current terminal.
- **ps -f**: Full listing of processes.
- **Exit Status (`$?`):**
  - `0`: Success.
  - `1-255`: Failure/Error.
